**Tweet 2 — Das Problem (konkret, keine abstraktion)**

```
ai agents can spend thousands of dollars.
they can sign transactions, deploy contracts, hold tokens.

but where do they store what they built?

github. which requires a human account.
ipfs. which goes offline.
your server. which you own, not them.

that's not an autonomous agent. that's a puppet with a very long string.
```

---

**Tweet 3 — Die Lösung (mechanismus, nicht marketing)**

```
inkd is a project registry on base.

one $INKD token locks to register a project.
0.001 eth per version push → permanent on arweave.
0.005 eth per ownership transfer → on-chain, provable, forever.

no accounts. no servers. no humans in the loop.

a wallet registers. a wallet owns. a wallet transfers.
```

**Tweet 5 — CLI (devs anschreiben)**

```
the cli is four commands:

inkd init       → register a project
inkd publish    → push a version (permanent, arweave)
inkd transfer   → transfer ownership (on-chain)
inkd info       → query any project

that's it.
if you can write a readme, you can use inkd.
```

**Tweet 7 — Honest Take (vertrauen aufbauen)**

```
real talk: inscribing project data on-chain isn't a new idea.

git has provenance. ipfs has content addressing. github has history.

what's new:
- ownership is a token, not an account
- versions are permanent, not mutable
- transfer is one transaction, not a password reset

we're not replacing github.
we're building the layer github can't be.
```
