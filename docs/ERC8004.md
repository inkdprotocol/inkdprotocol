# inkd + ERC-8004 Integration Guide

[ERC-8004](https://eips.ethereum.org/EIPS/eip-8004) is the Ethereum standard for Trustless AI Agents — co-authored by MetaMask, Ethereum Foundation, Google, and Coinbase. It defines three on-chain registries: Identity, Reputation, and Validation.

Notably, ERC-8004 explicitly excludes code and data ownership:

> *"Payments are orthogonal to this protocol and not covered here."*

inkd fills exactly this gap. Together, they form a complete agent stack.

---

## The Complete Agent Stack

| Layer | Standard | What it gives agents |
|-------|---------|---------------------|
| Payments | [x402](https://x402.org) | Ability to pay for APIs autonomously |
| Identity | [ERC-8004](https://eips.ethereum.org/EIPS/eip-8004) | Portable, verifiable on-chain identity |
| **Code + Data** | **inkd** | **Permanent, owned code registry** |
| Wallets | AgentKit / Privy / Safe | ETH, tokens, signing |

---

## How They Fit Together

An ERC-8004 agent identity points to a metadata URI. That URI can resolve to an inkd project — giving the agent a permanent, versioned, on-chain-proven record of what it built.

```
ERC-8004 Identity Registry
  └── agentId: 42 (ERC-721 token)
       └── tokenURI → https://api.inkdprotocol.com/v1/projects/77
                           ↓
                    inkd Project #77: "my-agent-v2"
                    Owner: 0x1234...abcd (= agent wallet)
                    Versions:
                      v1.0.0 → ar://QmXyz (initial code)
                      v1.1.0 → ar://QmAbc (update)
                      v2.0.0 → ar://QmDef (major release)
```

The ERC-8004 identity NFT proves WHO the agent is. The inkd project proves WHAT the agent built and OWNS.

## Setting Up the Integration

### 1. Register an inkd project for your agent

```typescript
import { wrapFetchWithPayment } from '@x402/fetch'
import { privateKeyToAccount } from 'viem/accounts'
import { base } from 'viem/chains'

const account = privateKeyToAccount(process.env.AGENT_PRIVATE_KEY as `0x${string}`)
const fetch = wrapFetchWithPayment(account, base)

// Register the agent's codebase on inkd
const res = await fetch('https://api.inkdprotocol.com/v1/projects', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name:          'my-agent-v2',
    description:   'AI agent that summarizes GitHub PRs',
    license:       'MIT',
    isPublic:      true,
    isAgent:       true,
    agentEndpoint: 'https://api.myagent.xyz/v1',
  }),
})

const { projectId } = await res.json()
console.log(`inkd project #${projectId} registered`)
```

### 2. Use the inkd project URI as your ERC-8004 tokenURI

When registering with an ERC-8004 Identity Registry, set the tokenURI to point to your inkd project:

```typescript
const tokenURI = `https://api.inkdprotocol.com/v1/projects/${projectId}`

// Call your ERC-8004 Identity Registry contract
await identityRegistry.write.register([tokenURI])
```

### 3. Keep the record updated

Every time your agent releases a new version, push it to inkd:

```typescript
// Push v2.0.0 automatically (e.g. in CI/CD)
await fetch(`https://api.inkdprotocol.com/v1/projects/${projectId}/versions`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    tag:         'v2.0.0',
    contentHash: `ar://${arweaveHash}`,
  }),
})
```

No human needed. The agent's wallet pays, the agent's wallet is the owner. Full autonomy.

## The ERC-8004 Agent Registration File

ERC-8004's Identity Registry stores a URI that resolves to an agent's registration file. With inkd, this file IS the on-chain project record. Here's the recommended structure:

```json
{
  "name": "my-agent-v2",
  "description": "AI agent that summarizes GitHub PRs in 3 bullets",
  "version": "2.0.0",
  "license": "MIT",
  "endpoint": "https://api.myagent.xyz/v1",
  "registry": {
    "inkd": {
      "projectId": "77",
      "projectUrl": "https://api.inkdprotocol.com/v1/projects/77",
      "owner": "0x1234...abcd",
      "latestVersion": "ar://QmDef..."
    }
  },
  "capabilities": [
    "summarize_pr",
    "code_review",
    "changelog_generation"
  ]
}
```

Host this file on Arweave and use the `ar://` hash as both the inkd `readmeHash` and the ERC-8004 tokenURI.

## Why This Matters

ERC-8004 solves "can I trust this agent?" through identity and reputation.

inkd solves "what has this agent built and who owns it?" through permanent code registration.

Together, they answer the two fundamental questions in an agent economy:
1. **Who is this agent?** → ERC-8004 Identity Registry
2. **What did this agent create?** → inkd Registry

Neither standard replaces the other. They're designed to be used together.
